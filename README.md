# Система учёта врачей и пациентов

## О проекте

Приложение предназначено для ведения картотеки врачей и пациентов. В системе можно создавать, редактировать, удалять и просматривать информацию о врачах и пациентах.
Связь между сущностями реализована по принципу: один врач может иметь нескольких пациентов.

---

## Выбранная архитектура

Для выполнения проекта был выбран вариант **Б - Fullstack Next.js**.

Выбор сделан потому, что такой вариант проще для небольшого учебного проекта:

- фронтенд и бэкенд находятся в одном приложении;
- используется один порт;
- не требуется отдельно настраивать CORS;
- удобнее запускать и развёртывать проект.

Серверная логика реализована через Route Handlers в директории `app/api`.

## Реализованный функционал

### Работа с врачами:

- просмотр списка врачей;
- создание врача;
- редактирование врача;
- удаление врача;
- просмотр карточки врача.

### Работа с пациентами:

- просмотр списка пациентов;
- создание пациента;
- редактирование пациента;
- удаление пациента;
- просмотр карточки пациента;
- выбор лечащего врача.

Дополнительно реализовано:

- поиск пациентов;
- фильтрация по врачу;
- пагинация.

## Сущности

### Doctor

Поля:

- id
- fullName
- specialization
- experience
- isAvailable
- gender
- email (необязательное)

### Patient

Поля:

- id
- fullName
- age
- diagnosis
- admittedAt
- isCritical
- doctorId
- notes (необязательное)

Связь:
один врач - несколько пациентов.

---

## API

### Врачи

GET `/api/doctors` — получить список врачей

GET `/api/doctors/[id]` — получить врача по id

POST `/api/doctors` — создать врача

PATCH `/api/doctors/[id]` — изменить данные врача

DELETE `/api/doctors/[id]` — удалить врача

---

### Пациенты

GET `/api/patients` — получить список пациентов

GET `/api/patients/[id]` — получить пациента по id

POST `/api/patients` — создать пациента

PATCH `/api/patients/[id]` — изменить пациента

DELETE `/api/patients/[id]` — удалить пациента

Все ответы возвращаются в формате JSON.
